# Professional, owner-manageable Gallery

> Project-based plan (`wp-content/docs/plans/`). Implemented 2026-07-21 — theme v2.12.0 / plugin v2.8.0.

## Context

The গ্যালারি page was the weakest surface on the site. `themes/bhela/page-templates/template-gallery.php:10-17` built the image list by `glob()`-ing theme folders (`assets/images/{hero,boat,cabins,spots,food}`), so the 17 photos were **hardcoded into the theme**: the owner could not add, remove, reorder, caption, or group them without FTP access, and a theme update could overwrite them. The page rendered one flat square grid with a minimal lightbox (open / close only).

Goal: the owner manages the gallery from WordPress like any other content, and the page looks like a premium tour operator's — filterable, captioned, masonry, with a real lightbox.

Owner decisions: a **dedicated Gallery admin menu** (not Customizer slots, not Gutenberg blocks), plus **category filter tabs, advanced lightbox, captions, masonry layout**.

## Design decisions

**Photos are a CPT in the plugin** (`bhela_gallery`), modelled on the `bhela_review` CPT in `plugins/bhela-booking/includes/reviews.php`. It lives in the plugin (not the theme) so photos survive a theme switch, matching where reviews and trips already live.

- `supports => array( 'title', 'thumbnail', 'page-attributes' )`. **The featured image is the photo** — reuses WordPress's native media picker, so no `wp.media` code is needed anywhere. Title = caption. Native "Order" field = sort order.
- Omitting `'editor'` means the CPT uses the **classic editor** — the owner sees only Title, Featured Image, Order, Category checkboxes.
- `show_in_menu => 'edit.php?post_type=bhela_booking'` — nests under the 🛶 Bookings menu like Reviews.

**Categories = taxonomy** `bhela_gallery_cat`, registered `hierarchical => true` so it renders **checkboxes** rather than the free-text tag box, which would invite duplicate Bangla terms from typos. Seed terms: কেবিন, খাবার, হাওর, রুফটপ. Tabs order by `term_id ASC` (taxonomies have no menu_order; alphabetical Bangla is arbitrary).

**Image sizes — not `bhela-card`/`bhela-wide`.** Both are **hard crops** (`themes/bhela/functions.php:28-29`, third arg `true`), which would force every thumb to 4:3 and defeat masonry. Use core **`medium_large`** (768w, soft crop, already generated) for thumbs and **`full`** for the lightbox.

**Masonry via CSS `column-count`**, not grid row-span. Row-span needs JS to measure heights, recomputed on every lazy-image load, filter toggle and resize — fragile, failure mode is overlapping images. Multi-column reflows natively on `display:none` and on image load with zero JS.
*Accepted trade-off:* multi-column fills column-major, so visual order won't match the "Order" field exactly.

**Seeding: a one-click import button, not automatic.** Auto-importing 17 JPEGs on `admin_init` would run `wp_generate_attachment_metadata()` 17× — ~120 derivatives in one blocking request, a realistic 504 on shared hosting. Instead an admin notice on the empty gallery list offers an import button. **Idempotency by seed key** (`_bhela_gallery_seed_key`, e.g. `spots/spot-3.jpg`) stored on both post and attachment — clicking twice, or reactivating the plugin, is a no-op.

**The page never breaks:** if the plugin is inactive or no photos are published, the template falls through to the existing `glob()` block.

## Changes

**Created `plugins/bhela-booking/includes/gallery.php`** — CPT + taxonomy, admin columns (thumbnail / category / order), `bhela_bm_get_gallery()`, `[bhela_gallery]` shortcode, term seeder, `bhela_bm_import_theme_image()` and the `admin_post_` import handler (nonce + `manage_options`).

`bhela_bm_import_theme_image()` **ports the body** of `themes/bhela/functions.php:538-565` rather than calling it — a plugin must not depend on a theme function, and that helper returns nothing and skips when a thumbnail already exists.

Markup contract (one code path for new and legacy markup):
```html
<a class="bhela-gallery__item" href="FULL_URL" data-cats="cabin food" data-caption="…"><img …></a>
```
Thumbs via `wp_get_attachment_image( $id, 'medium_large', false, array( 'loading' => 'lazy', 'decoding' => 'async' ) )` — WP emits `width`/`height`, so with `height:auto` the browser reserves the box before load (anti-CLS). Tab bar renders only when ≥2 terms are in use.

**Modified**
- `plugins/bhela-booking/bhela-booking.php` — require gallery.php; version bump.
- `themes/bhela/page-templates/template-gallery.php` — use the shortcode when it returns non-empty, else the original glob block verbatim.
- `themes/bhela/assets/js/theme.js` — replaced the old lightbox with an `initGallery()` module.
- `themes/bhela/style.css` — masonry, tabs, captions, upgraded lightbox. **`.gallery-grid` rules kept intact** for the fallback; new markup uses `.bhela-gallery` because `.gallery-grid a` forces `aspect-ratio: 1`.
- `themes/bhela/functions.php` — `BHELA_VERSION` bump (the only cache-bust for `style.css` **and** `theme.js`).
- `docs/BHELA-Owner-Manual.md` — new Gallery section.

### Frontend behaviour
- JS reads a **data-attribute contract off the anchors**, never a container class, so legacy markup takes the same path: `caption = a.dataset.caption || img.alt || ''`, `cats = (a.dataset.cats || '').split(/\s+/)`. Legacy anchors yield `cats = []` → lightbox, no tabs.
- `items[]` built once; `visible[]` derived inside `applyFilter()`; `index` indexes **`visible`**, so prev/next cycle only the filtered set and the counter reads `(index+1) / visible.length`.
- Preload next image; swipe requires `|dx| > 45 && |dx| > |dy| * 1.5`; single keydown handler gated on `is-open`.

### CSS essentials
```css
.bhela-gallery { column-count: 3; column-gap: 1rem; }
@media (max-width: 900px) { .bhela-gallery { column-count: 2; } }
@media (max-width: 560px) { .bhela-gallery { column-count: 1; } }
.bhela-gallery__item { break-inside: avoid; display: block; width: 100%; }
.bhela-gallery__item[hidden] { display: none; }   /* REQUIRED */
.bhela-gallery__item img { width: 100%; height: auto; display: block; }
```
The `[hidden]` rule is **mandatory** — the UA stylesheet loses to the author-level `display:block`, so without it filtering silently does nothing.

### Accessibility
`role="dialog"`, `aria-modal`, Bangla `aria-label`s; focus saved/restored; Tab trapped across the three buttons; `body overflow:hidden` while open; caption + counter in one `aria-live="polite"` region; filter tabs are `<button aria-pressed>` (not `role="tab"`); thumbs stay real links so the page degrades without JS.
